HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about an element.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • The base element defines the base URL for a webpage.
  • Child elements are nested inside other elements.
  • Semantic HTML elements describe it's content and the role it plays on a webpage.
  • div element creates a division in page.
  • span element creates an inline division.
  • br element creates a line of empty space, or a line break.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • 3 types of CSS: Inline, Internal, and External Style Sheet.
  • Declarations: Contain two important components, the CSS property we want to apply and the value of the property.
  • * selector: special wild card, applies a rule to all elements visible on the page.
  • text-align: one way to center a title or heading on a webpage.
  • display property is assigned with the block value, which assigns each image to start on a new line.
  • Class attribute: Allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • Any HTML element assigned to the class card will gain these property assignments from the CSS.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • add command, adds modifications in the current working branch to the staging area.
  • -A flag indicates that we want to add all the changes.
  • pull command receives a branch's modifications into the local environment.
  • origin keyword indicates the source of the pull will be in the GitHub repo.
  • main keyword indicates the main branch.
  • Always do a git pull before you start working to make sure you're working on the most updated code.
  • push, pushes change.
